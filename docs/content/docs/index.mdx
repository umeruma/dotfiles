## install (macOS / Linux)
This dotfiles repository is designed to be installed at `~/Codes/dotfiles`.

Review the contents of the script before running it:  
[https://dot.umeru.ma/install](https://dot.umeru.ma/install)

Open Terminal.app, and run:

```
bash -c "$(curl -fsSL https://dot.umeru.ma/install)"
```
This will clone the repository to `~/Codes/dotfiles` and set up your dotfiles. If the repository already exists, it pulls the latest changes before continuing.

After cloning, the installer installs mise (if needed) and runs [`mise bootstrap`](https://mise.jdx.dev/bootstrap.html) to converge OS packages, login shell, and `mise run deploy`. Details: [mise bootstrap](/mise-bootstrap).

Notes on bash and curl options:

bash:
- `-c`: Execute the given string as a bash command (e.g., `bash -c "echo hello"` outputs `hello`).

curl:
- `-f` (`--fail`): Fail silently on server errors (no output, exit non-zero).
- `-s` (`--silent`): Silent mode — do not show progress meter or error messages.
- `-S` (`--show-error`): When used with `-s`, show error messages on failure (since `-s` alone is fully silent).
- `-L` (`--location`): Follow HTTP redirects automatically.

## jump to dotfiles

In Terminal, you can quickly jump to the dotfiles directory by running:
```
dot 
```

.zshenv & .zshrc contains:
```
# on .zshenv
export DOTFILES="$HOME/Codes/dotfiles"

# on .zshrc
alias dot='cd $DOTFILES' 
```

## Deploy dotfiles

To deploy your dotfiles to your home directory, simply run:

```
dot && mise run deploy
```

### How it works

Symlinks are managed declaratively by mise's `[dotfiles]` sections in [`mise.dotfiles.toml`](https://github.com/umeruma/dotfiles/blob/main/mise.dotfiles.toml) (cross-platform; loaded via `env = ["dotfiles"]` in `.miserc.toml`) and [`mise.macos.toml`](https://github.com/umeruma/dotfiles/blob/main/mise.macos.toml) (macOS-only). Each entry is an explicit allowlist item:

```toml
[dotfiles]
"~/.zshrc" = { source = "home/.zshrc", mode = "symlink" }
```

Directories that receive runtime writes (logs, state — e.g. `~/.config/karabiner`, `~/.config/micro`, `~/.config/lazygit`) are linked **per file**, so app state stays local and never leaks into the repo. Purely static config dirs (`~/.config/doom`, `~/.config/ghostty`, `~/.config/sheldon`) are linked as whole directories.

Useful commands:

```
mise dotfiles status   # show applied / differs / missing per entry
mise run clean         # remove every symlink pointing into home/ (incl. dangling)
dottrack <path>        # start tracking an existing file (adds entry + relinks)
```

Agent skills (`home/.agents/skills/`) are a single source of truth shared by all agents: `~/.agents` is one whole-dir link (Cursor and GitHub Copilot read `~/.agents/skills/` natively), and Claude Code gets per-skill links in `~/.claude/skills/`. The whole-dir link is deliberate: skills installed with `npx skills` and its `.skill-lock.json` land inside the repo through the link, so installs are tracked by git.

## macOS defaults

macOS preferences are applied by `mise bootstrap` from [`mise.macos.toml`](https://github.com/umeruma/dotfiles/blob/main/mise.macos.toml) (`[bootstrap.macos.*]` and `[bootstrap.macos.defaults]`). Re-run bootstrap to converge, or inspect drift:

```
dot && mise bootstrap macos defaults status
```

Details: [mise bootstrap](/mise-bootstrap).
